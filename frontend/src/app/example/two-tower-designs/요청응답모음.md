# onboarding-service 요청/응답 모음

기본 공개 API는 아래 4개다.

```text
GET /health
GET /surveys/active
POST /surveys/active/preview
GET /surveys/results/{profile_code}
```

로그인 필요 API는 아래 2개다.

```text
PUT /surveys/me/profile
GET /surveys/me/profile
```

기준 베이스 URL 예시:

```text
http://localhost:8088/api/onboarding
```

## 1. 헬스체크

### 요청

```http
GET /health
```

### 응답

```json
{
  "ok": true,
  "service": "onboarding-service",
  "version": "0.1.0",
  "model_id": "onboarding_two_tower",
  "trained_at": "2026-06-21T11:22:33.456789+00:00",
  "item_count": 40
}
```

## 6. 로그인 사용자가 저장된 결과 불러오기

### 요청

```http
GET /surveys/me/profile
Authorization: Bearer <access_token>
```

또는

```http
GET /surveys/me/profile?top_k=8
Authorization: Bearer <access_token>
```

### 응답

응답 shape는 `PUT /surveys/me/profile`와 같다.

## 7. 프론트에서 제일 많이 쓰게 될 값

설문 화면:

- `questions[].id`
- `questions[].prompt`
- `questions[].selection_type`
- `questions[].max_selections`
- `questions[].options[].code`
- `questions[].options[].label`

결과 화면:

- `profile.profile_code`
- `profile.user_profile.profile_name`
- `profile.user_profile.preferred_category_code`
- `prediction.recommendations[].rank`
- `prediction.recommendations[].score`
- `prediction.recommendations[].area_name`
- `prediction.recommendations[].service_category_name`
- `prediction.recommendations[].area_profile_type`

회원 저장 흐름:

- `survey_response_id`
- `profile.profile_code`
- `profile.auth_user_uuid`
- `profile.updated_at`

## 8. 숨겨진 경로

아래 경로는 기본 운영 설정에서 외부에 안 열린다.

```text
/two-tower/catalog
/two-tower/predict
/two-tower/evaluation
/two-tower/train
/two-tower/profiles/users/{auth_user_uuid}
/two-tower/profiles/code/{profile_code}
```

디자인 작업은 이 경로들을 기준으로 하지 말고 `surveys/*` 기준으로 하면 된다.
