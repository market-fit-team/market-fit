사용자가 너에게 지목한 designX 폴더에 가서 작업하기. 너는 그 폴더만 작업 가능함.

기본적인 흐름은
사용자가
designX 에서 설문을 입력하면
designX/res 에서 응답을 보는데
res에서 응답이 달라짐!

그리고 작업할 때 필요한 파일들은
designX/\_components 안에 만들면 되고
README.md 에는 너가 어떤 모델명인지를 포함한 설명 적으면 됨

다른 폴더는 너가 열람할 수 없고
단순히
/nginx-msa/frontend/src/app/onboarding 랑
/nginx-msa/frontend/src/shared/components/ui 만 볼수 있음
nginx-msa/frontend/src/shared/components/ui 의 컴포넌트들을 임포트해서 사용하는건 허용!

글구 API 연동은 안하고 니가 목데이터로만 할거임!
designX/\_fixtures 폴더 만들어서 사용해도 됨!!
/nginx-msa/frontend/src/app/example/two-tower-designs/질문지.md
/nginx-msa/frontend/src/app/example/two-tower-designs/요청응답모음.md
이거 토대로 작업하는거임
